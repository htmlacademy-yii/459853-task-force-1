Классы
==========================
attachments
  класс будет создавать объект для записи в БД.
  Загрузка в файловую структуру, перемещение в нужную директорию, переименование (возможно).
  Затем должен будет возвращать id в другой класс.
==========================
Tasks
  Создает и хранит информацию об объекте "Задача".
  Создавший пользователь сможет редактировать задачу.
  Будет хранить информацию о тексте, постановщике, времени окончания, сумме и других полях.
  Будет возможность возвращать список задач.

  Пользователь будет логиниться, и сможет создавать задачи, редактировать или откликаться на них.
==========================
Users
  Создание и хранение информации о пользователе его роли в системе.
  Будет создание, редактирование и хранение.

==========================
Comments
  Создание и хранение объекта о комментарии, кто оставил, кому оставили, оценка, текст.

  Только оставивший пользователь будет иметь возможность редактировать.
==========================
Catergory
  Отвечает за создание и редактирование категорий. (скорее всего должен быть доступен только админу).
  будет возвращать информацию о категориях, для других классов. (Tasks)
==========================
Notification
  будет вызывать отправку уведомлений пользователям в зависимости от статуса задания.
  Формировать email по шаблонам, отправка.
==========================
statuses
  класс для хранения и создания статусов (тут тоже наверно должен быть доступен только админу)
==========================
specializations
  Отвечает за создание и редактирование специализаций для исполнителей. (скорее всего должен быть доступен только админу).
  будет возвращать информацию о специализациях, для других классов. (Users)
==========================
task_response
  создание и хранение для откликов на задания,
  пользователь должен быть с ролью "исполнитель"
==========================
